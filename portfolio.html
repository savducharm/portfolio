Portfolio Site

Designing and Coding Website

The Challenge

The goal of this project was to create a personal website to host my project case studies, provide a way for potential employers to contact me, and develop an online presence and brand.

Moreover, I wanted to expand upon my small amount of coding knowledge to be able to communicate effectively with developers in the future.

Target Base

  • Recruiters
  • Hiring Managers
  • Fellow UX Designers

Tools Used

  • Atom
  • GitHub
  • Adobe XD

The Process

For my approach, I followed Design Thinking and Iterative design processes. However, given the type of project, not all stages were executed fully.

Step 1

Content

As the general layout and sitemap was given in the course, the first step of this project was to decide on the personal information and projects I would include on the site.

Writing the Copy

I began by writing the copy for each page to help create the voice of my site and to help me determine what content I wanted to include.

I first wrote case studies for my two most recent projects, establishing a reusable outline in which to document any future case studies I may want to add.

With the content for the work sections fleshed out, I then wrote up a biography for my About page and decided on what other personal information to include.

Step 2

Visual Design and Planning

With all the copy written, I was able to form a better idea of the structure and flow of content. The next step was to create the visual identity of the site and my personal brand.

Creating a Personal Brand

I used a logo I created for myself previously as the jumping off point, using it to determine header and paragraph fonts and styles.

From there, I gathered all supporting visual elements to supplement the text. I was inspired by the self-portrait I chose for my About page to determine the siteâ€™s color scheme of deep greens and pastel highlights. I felt these matched with the elegance of my logo and moreover my own love of nature and down to earth personality.

Step 3

Creation and Iteration

With my copy written, content curated, and thematic elements chosen, it was time to begin building my first iteration.

Mobile-First Design

Using the basic site structure provided by CareerFoundry, I created the mobile version of my site using the HTML and CSS I was learning. I first used HTML to establish the structural elements and general layout of the site. Then, I integrated CSS to style and personalize the site by using my chosen color scheme, fonts, and copy.

Responsive Design

Once the mobile site was finished, I added media breakpoints for tablets and desktop screen sizes. From there, I modified the mobile code to optimize the site at these various screens, ensuring it would look good from any device.

Next, I added animations and interactions to the site. I did this using both CSS and JavaScript to add hover states, independent animations, and a drop-down menu.

Learning from Mistakes

All together, this third step took the longest as there was constant back and forth and subtle changes to test out. As it was my first time coding an entire site, I experienced many mistakes and hiccups. However, I learned an incredible amount about coding and industry best practices. By the end of this step I had created a well-functioning and rather aesthetic site, if I do say so myself (and I do!).

Step 4

Testing with Real Users

With a completed site, the last step was to put it in front of real users to determine its usability and possible pain points.

Preparing for Testing

This was a very crucial step to help bring to light any severe usability problems before publishing the site and sharing it with potential employers. I would not be an attractive UX job candidate if my own portfolio site had any major UX issues.

To begin, I first establish four goals for the usability test, listed below.

I then established a set of criteria for my target users. Given it would have been very difficult to test with actually recruiters and UX employers, I settled on those listed below.

Next, I wrote a usability test script, which can be viewed here, and recruited 5 participants who I felt fit my target user criteria.

Usability Test Goals
  • Ensure my portfolio site is easy to navigate and visually captivating.
  • Ensure my projects are easy to access and review.
  • Ensure users are able to find my resume.
  • Ensure users are able to find how to contact me.
Target Participant Criteria
  • Between 25-60 years old
  • Technically savvy on phones and computers
  • Familiar with UX/UI design

Results

Overall, the majority of participants did not experience any critical usability issues. All expressed that the site was very professional and clean.

That said, some concerns were raised around the expectations of the navigation header links and image sizes. So, I created a usability test report, documenting the main usability issues and my recommendations for future iterations.

The full usabulity report can be reviewed here.

Key Usability Issues
  • Contact info in footer not intuitive (medium severity)
  • Work link in navigation misleading (medium severity)
  • Projects' visual assets too small to see clearly (high severity)
Actionable Steps
  • Put contact information on separate page or make footer height of viewport.
  • Have work link jump to first project or create separate work page.
  • Enable users to increase size of images to better see details.
Step 5

Implementing Solutions

In this final step, I sought to find solutions to the main usability issues found in my user testing.

Contact Page

Observation

All participants were expecting a separate page for the Contact information and were generally dissatisfied by being sent to the footer for the information.

Severity: 2

Recommendation
  • Creata a separate Contact page or make the footer take up the entire viewport for similar effect.
Solution

Increased the height of the footer to take up more the viewport, thus giving the illusion of another page.

Work Link

Observation

4 participants expressed dissatisfaction with the Work link, expecting it to send them to another page (rather than the home page). Participant 1 said he expected to see a work history on the Work page and sought my resume here rather than the About page (as did Participant 2).

Severity: 2

Recommendation
  • Having the Work link jump users to the first project listed, as Participant 4 suggested.
  • Listing out more work history information on the About page, as participant 1 suggested.
Solution

Changed Work link in navigation to jump users to the first listed project.

Project Visuals

Observation

Another concern that was raised by 3 of the 5 participants was the inability to see and read the details on many of the QuickCare project visuals. Participant 2 expressed that it may cause her to lose interest.

Severity: 3

Recommendation
  • Enable users to either click images to enlarge them or adding a hover state that grows the images so users can more easily read the details of each image.
Solution

Added image into a modal that pops up when users click specific images.

Added links to certain assets for users who may still want more information.

Cleaning Up the Code

Next, I used code linters to ensure clean, consistent, and readable code. Then, I conducted cross-browser and cross-device testing to ensure all users would be experiencing the same thing. Some issues were discovered, mainly visual inconsistencies, and promptly fixed.

Accessibility

Lastly, I checked my site for accessibility by first confirming all images had alternative text. I then checked color contrasting to ensure all users could easily read. This led me to change some of my colors to darker tints and modify some opacities.

Looking Ahead

Further Improvements

Although I feel I have accomplished quite a feat here, I do have more that I would like to add and expand upon.

Going forward, there remains much to be done to here.

The main improvements I would like to improve upon are:

  1. Creating a separate page for my work that allows users to filter projects by type.
  2. Adding a process page that goes into more depth of how I approach and attempt to solve problems in my work.
  3. Continue to add more projects!

Conclusions

My main goal was to create a personal website to host my project case studies, provide a way for potential employers to contact me, and develop an online presence and brand. I feel that was achieved, as I now have a published site meeting these three criteria. I feel confident that this site shows off the skills I have acquired during my UX Immersive course through CareerFoundry and as a designer.

Additionally, I wanted to expand upon coding knowledge to better communicate with developers in the future. I am very impressed with how much I have learned and the sheer feat of coding this site myself in such a short time span. I greatly look forward to applying my new coding knowledge in my future work, as well as expanding upon my portfolio site.
